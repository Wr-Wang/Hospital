# 设计系统：医院管理系统

> **适用范围**：本设计规范仅涵盖 `Hospital.App`（WPF 桌面客户端）的 UI 视觉与交互设计。
> `Hospital.Api`（Web API）无前端界面，不在本文档范围内。
> 相关文档：[WPF_ARCHITECTURE.md](docs/WPF_ARCHITECTURE.md)、[WPF_UI_INVENTORY.md](docs/WPF_UI_INVENTORY.md)

## 1. 视觉主题与氛围

专业的医疗级桌面应用，采用清晰的分层信息架构。深青色侧边栏固定导航，白色内容区最大化临床数据录入的可读性。整体风格**亲和、沉稳、高效**——优先保证清晰度和长时间值班使用时的视觉疲劳控制。

- **密度**：适中——舒适的间距与紧凑的数据表格并存
- **氛围**：临床感、可信赖、克制
- **设计参考**：现代企业级管理后台（Ant Design Pro、Element Plus）风格适配 WPF 桌面端
- **仅浅色模式**（深色主题不在当前范围）

## 5. 布局原则

### 间距刻度

基础单位：4px。常用间距：2、4、8、12、16、20、24、28、32。

| 间距 | 用途 |
|------|------|
| 4px | 紧凑内边距 |
| 8px | 相邻元素间距、网格间距 |
| 12px | 标签与输入框间距、图标与文字间距 |
| 16px | 卡片水平内边距、表单按钮间距 |
| 20px | 卡片内边距、分层区域间距 |
| 24px | 区块间距、页面水平边距 |
| 28px | 页面上下边距、登录表单向内边距 |
| 32px | 页面边距、登录卡片内边距 |

### 页面结构

```
+-- 侧边栏 (260px) --+-- 顶栏 (自动高度) ---------+
|                      |                             |
| 用户头像 + 信息       | 面包屑             用户徽章 |
|                      |-----------------------------|
| 导航项 (ListBox)     | 内容区 (填满剩余空间)       |
|                      |                             |
|                      |   (需要时显示 ScrollViewer)  |
| 退出系统按钮         |                             |
+----------------------+-----------------------------+
```

### 网格系统

- 仪表板统计卡片和功能链接使用三列等宽网格（列间距 8px）
- 表单采用单列布局（标签在上、输入框在下）
- 表单按钮行采用双按钮布局（次按钮 + 主按钮，间距 16px）

### 留白理念

区块间使用充足留白（24px），分组控件内使用紧凑间距。表单整体感觉开阔且易于扫描。仪表板卡片通过一致的圆角和背景形成视觉连接。

## 6. 深度与层次

三种表面层级：

| 层级 | 元素 | 样式 |
|------|------|------|
| **基底** | 页面背景、侧边栏 | 平面、无阴影 — `#F0F2F5` / `#0F766E` |
| **凸起** | 卡片、统计面板、表单区块 | 背景 `#FFFFFF`，通过背景对比形成层次，内容区无阴影 |
| **浮升** | 模态弹窗、登录卡片 | 背景 `#FFFFFF` + `DropShadowEffect (BlurRadius 24, Direction 270, Opacity 0.12, ShadowDepth 4, Color #000000)` |
| **覆盖** | 加载遮罩 | 半透明 `#80FFFFFF` 覆盖在内容之上 |

登录卡片使用显著的阴影形成视觉焦点。页面内卡片依靠与 `#F0F2F5` 背景的对比而非阴影来区分层次。

## 7. 做与不做

### 要做
- 所有主要操作和聚焦指示器使用 `#14B8A6`
- 页面主背景使用 `#F0F2F5`——永远不要用纯白色作为页面背景
- 侧边栏固定 260px 宽度，内容区填满剩余空间
- 始终使用 4px 基准间距体系
- 表单标签始终显示在输入框上方（不要仅使用占位符作为标签）
- 验证消息使用 `#FEF2F2` 背景的错误横幅
- 异步操作显示加载遮罩（半透明白色）
- 卡片圆角 10px，按钮圆角 6px，标签/徽章圆角 4px

### 不要做
- 不要使用纯黑色 `#000000` 作为文字颜色——最高对比度使用 `#111827`
- 不要在页面内卡片上添加阴影——依靠背景对比区分层次
- 不要在彩色背景上使用彩色文字，确保足够的对比度
- 不要在导航中使用纯图标按钮而不带文字标签
- 不要将表单输入框拉伸超过 400px 宽度
- 不要使用需要外部加载的自定义字体——仅使用系统字体
- 不要在模态弹窗内再打开模态弹窗
- 不要在可滚动区域内嵌套可滚动区域

## 8. 响应式行为

本应用为 WPF 桌面应用，无浏览器断点。关键窗口约束：

| 约束 | 值 |
|------|------|
| 最小窗口尺寸 | 1024 × 600 |
| 默认窗口状态 | 最大化 |
| 登录窗口 | 固定尺寸，SizeToContent="WidthAndHeight"，禁止调整大小 |
| 侧边栏 | 固定 260px，不可折叠 |
| 内容区 | 使用 ScrollViewer 填满剩余空间 |

### DPI 感知

- PerMonitorV2 DPI 缩放（通过 app.manifest 启用）
- 所有尺寸使用设备无关像素
- 文字缩放跟随系统 DPI

### 内容溢出

- 仪表板页面：`ScrollViewer` 垂直滚动
- 数据表格：水平 + 垂直滚动
- 选项卡内容（门诊诊间）：填满可用空间，各选项卡独立滚动
