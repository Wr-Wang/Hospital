# 微信小程序开发方案

## 一、背景概述

当前系统已实现一套完整的**医院信息管理系统（HIS）**，覆盖以下业务域：

| 业务域 | 功能 |
|--------|------|
 | **基础数据** | 院区、科室、职工、字典管理 |
| **患者管理** | 患者建档、重复检测、合并、360视图 |
| **门诊排班** | 医生排班、时段管理、配额控制 |
| **挂号管理** | 窗口挂号、退号、队列分配 |
| **医生工作站** | 接诊队列、病历书写、诊断、开药、开检查检验 |
| **收费管理** | 待缴费项查询、统一支付（药品/检查/检验） |
| **药房管理** | 发药、退药、库存管理 |
| **系统管理** | 用户、角色、权限、审计日志 |

**现有客户端**：WPF桌面应用（面向医院内部工作人员）

**小程序定位**：面向**患者/居民**的移动端服务入口，与现有后端API无缝对接。

## 二、小程序功能规划

### 2.1 首页与服务导航

- 医院简介与科室导航
- 公告/通知展示
- 快捷入口：预约挂号、门诊缴费、报告查询、排队进度

### 2.2 预约挂号

- 科室列表（按院区筛选）
- 医生排班查看（日期 + 时段）
- 号源选择与确认
- 患者选择/添加就诊人
- 预约提交与确认
- 我的预约记录（待就诊/已就诊/已取消）

### 2.3 门诊缴费

- 待缴费列表（药品/检查/检验）
- 在线支付（微信支付）
- 缴费历史查询
- 支付详情

### 2.4 报告查询

- 检验报告列表及详情
- 检查报告列表及详情
- 报告解读说明

### 2.5 就诊记录

- 历史就诊记录
- 诊断信息
- 处方/用药记录
- 病历摘要

### 2.6 排队进度

- 当前挂号队列实时进度
- 候诊人数显示
- 过号提醒

### 2.7 个人中心

- 就诊人管理（绑定/解绑）
- 实名认证（身份证 + 人脸）
- 我的订单
- 消息通知
- 设置与帮助

## 三、技术方案

### 3.1 整体架构

```
┌─────────────────────────────────────────────────┐
│              微信小程序 (WeChat Mini Program)      │
│  ┌──────────┐ ┌──────────┐ ┌────────────────┐   │
│  │ WXML/WXSS │ │  Page/Comp │ │  utils/api.js  │   │
│  └──────────┘ └──────────┘ └────────────────┘   │
└──────────────────────┬──────────────────────────┘
                       │ HTTPS (JWT Token)
                       ▼
┌─────────────────────────────────────────────────┐
│           Hospital.Api (ASP.NET Core Web API)     │
│  ┌─────────────────────────────────────────────┐ │
│  │        Controllers (已有)                   │ │
│  │  Patient / Schedule / Registration /        │ │
│  │  Encounter / Prescription / LabOrder /      │ │
│  │  Cashier / Diagnosis / MedicalRecord / ...  │ │
│  └─────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────┐ │
│  │     新增：MiniProgramController             │ │
│  │  └ 小程序专用聚合接口（减少客户端请求次数）   │ │
│  └─────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

### 3.2 与现有后端的关系

- **复用现有 API**：绝大部分功能可直接调用已有 Controller 接口
- **新增小程序专用接口**：对部分场景做聚合（如首页数据聚合），减少客户端请求次数
- **认证复用**：小程序使用微信登录（wx.login）获取 code，后端对接微信接口换取 openid，签发自定义 JWT Token
- **微信支付对接**：新增微信支付回调处理

### 3.3 技术选型

| 层 | 技术 |
|----|------|
 | 前端框架 | 微信原生开发 / 或 UniApp（可选） |
| UI 组件库 | WeUI / Vant Weapp |
| 后端对接 | 复用现有 ASP.NET Core API |
| 扩展开发 | C#（新增 Controller 和 Service） |
| 认证 | 微信 OAuth + JWT |
| 支付 | 微信支付 JSAPI |
| 推送 | 微信订阅消息 |

### 3.4 建议项目结构

```
hospital-weapp/
  pages/
    index/                  -- 首页
    registration/           -- 预约挂号

      result/               -- 支付结果

      lab/                  -- 检验报告
      rad/                  -- 检查报告
    records/                -- 就诊记录
      detail/               -- 就诊详情
    queue/                  -- 排队进度
    profile/                -- 个人中心

```

## 四、后端新增工作

### 4.1 新增 Controller

| 控制器 | 说明 |
|--------|------|
 | `MiniProgramAuthController` | 微信登录、手机号获取、用户绑定 |
| `PaymentCallbackController` | 微信支付回调通知处理 |

### 4.2 新增 Application Service

| Service | 说明 |
|---------|------|
 | `IWeChatAuthService` | 微信登录与 openid 管理 |
| `IWeChatPaymentService` | 微信支付统一下单、退款、回调 |

### 4.3 现有 API 适配

- 部分接口需增加患者端数据权限过滤（仅查询当前患者数据）
- 已有接口（排班查询、科室列表等）可直接复用

### 6.4 业务确认

| 序号 | 事项 | 说明 |
|------|------|------|
 | 1 | 是否支持**在线建档**（患者通过小程序自助建档）或仅支持绑定已有档案 |
| 2 | **实名认证方式**：身份证号 + 姓名（基础）/ 人脸识别（增强） |
| 3 | 是否开放**所有科室**的在线预约，或仅限部分科室 |
| 4 | **预约规则**：提前几天开放号源？是否支持当天预约？ |
| 5 | **退号规则**：就诊前多久可退？退费流程？ |
| 6 | **支付场景**：仅支持门诊缴费？是否包含住院预缴？ |
| 7 | **报告查询范围**：仅检验报告？是否包含影像报告/胶片下载？ |
| 8 | 是否需要**消息推送**（预约成功提醒、排队提醒、报告出具提醒） |
| 9 | 是否对接**院内 HIS 系统**？或当前系统即为正式生产系统？ |
| 10 | **部署环境**：现有 API 部署在内网还是公网？是否需要改造网络策略？ |

## 七、建议实施路径

```
Phase 1 (MVP) ─── 预约挂号 + 排队进度 + 个人中心
  │
Phase 2 ───────── 门诊缴费（微信支付）+ 就诊记录查看
  │
Phase 3 ───────── 报告查询 + 消息推送
  │
Phase 4 ───────── 优化迭代（在线建档、人脸认证、住院服务等）
```

**建议第一期先实现预约挂号和排队进度**，这是患者最核心的需求，也是微信小程序最常用的就医场景。

## 八、风险与注意事项

1. **微信小程序审核**：医疗类小程序需要提供《医疗机构执业许可证》等资质文件，审核周期较长，建议提前准备
2. **数据安全**：患者医疗数据属于敏感个人信息，需确保 HTTPS 传输、Token 有效期管理、数据脱敏等安全措施
3. **号源同步**：如果同时存在窗口挂号和在线预约，需要做好号源的实时同步，避免超卖
4. **网络策略**：现有 API 若部署在内网，需通过反向代理（Nginx）或 VPN 对外暴露小程序可访问的 HTTPS 接口
5. **微信支付费率**：医疗类微信支付有优惠费率政策，需提前申请
