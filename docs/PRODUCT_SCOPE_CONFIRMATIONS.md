# 产品范围与需求确认书

本文档落实《医院管理系统 — 产品需求清单（待确认）》中待拍板项，作为研发与测试的**需求基线**。若与商务合同不一致，以合同为准并回写本文档版本号。

**文档版本**：1.0  
**基线说明**：覆盖需求清单第 1–13 节全部模块；第 14 节分期为交付排期参考，不改变需求条目完整性。

---

## 1. 服务范围与终端（对应清单 §15 之 1、4）

| 确认项 | 结论 |
|--------|------|
| 门急诊与住院 | **门诊 + 急诊 + 住院** 全纳入同一产品基线。 |
| 门诊医生站 / 诊间 | **纳入**：含 PC 诊间医生站（清单 §5）。 |
| 管理端 | **纳入**：面向挂号、药房、设备科、护理部、财务、信息科等的 **Web 管理端**。 |
| 护士站 / 移动护理 | **纳入**：与住院护理（清单 §9）、监护（§8）衔接的护士工作台能力（可分期上线终端形态）。 |

---

## 2. 病人监护场景（对应清单 §15 之 2、§8）

| 场景 | 是否纳入 | 说明 |
|------|----------|------|
| A 门诊/普通住院观察 | 是 | 生命体征、护理记录、危急值提醒与闭环。 |
| B ICU/CCU 重症监护 | 是 | 床旁设备采集、评分表、报警分级与医护闭环；设备经中间件/厂商接口对接。 |
| C 远程监护 | 是 | 院外数据上报、患者绑定、医护审阅与预警；遵循互联网诊疗及数据安全合规。 |

**结论**：**A + B + C 组合**，不设三选一。

---

## 3. 收费与医保（对应清单 §15 之 3、§11）

| 确认项 | 结论 |
|--------|------|
| 费用引擎 | **纳入**：挂号、诊疗、药品、项目计费；退费/红冲规则与清单 §11 一致。 |
| 发票与票据 | **纳入**：电子发票/票据**接口位**及与院方财务系统对接规范。 |
| 医保 | **纳入**：读卡/电子凭证、目录对照、结算、对账；事前事中审核**接口位**；地区差异以规则包/配置区分。 |
| 「首期」说明 | 研发首期需具备**可对接真实医保与发票的联调环境**；具体省市接口以采购合同与监管为准，不改变产品能力范围。 |

---
